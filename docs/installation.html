Installation

Get up and running in about an hour. Start with GSC only — add other sources once that works.

Prerequisites

You need Python 3.8 or later. Check your version:

python3 --version
On Windows, use WSL (Windows Subsystem for Linux) for the smoothest experience.

Step 1 — Clone the repo

git clone https://github.com/kenneththapelo29/SEO-Command-Centre-.git
cd SEO-Command-Centre-

Step 2 — Install dependencies

pip install -r requirements.txt

Step 3 — Google Cloud project

1

Create a project

Go to console.cloud.google.com/projectcreate and create a new project named seo-command-center.

2

Enable APIs

Go to the API Library and enable:

- Google Search Console API

- Google Analytics Data API

3

Create a service account

Go to IAM and Admin → Service Accounts → Create.

Name: seo-command-center. Skip the optional permission steps.

4

Download the key

Click the service account → Keys tab → Add Key → Create new key → JSON. Save the downloaded file as service-account-key.json in the project root.

Never commit service-account-key.json to git. It is already in .gitignore but double-check before pushing.

Step 4 — Grant property access

Copy the service account email (looks like seo-command-center@your-project.iam.gserviceaccount.com).

GSC: Search Console → Settings → Users and permissions → Add user → paste email → Full access.

GA4: Analytics → Admin → Property access management → + → paste email → Viewer.

Step 5 — Configure

Edit config.json with your site details. See the Configuration guide for all options.

Step 6 — First fetch

python run_fetch.py --sources gsc,ga4

You should see JSON files appear in data/gsc/ and data/ga4/.

If you see checkmarks and row counts — you are set up. Open Claude Code in the project directory and start asking questions.

Common errors

"The caller does not have permission" — the service account email was not added to the GSC or GA4 property. Go back to Step 4.

"File not found: service-account-key.json" — the key file is not in the project root directory. Move it there.

"ModuleNotFoundError" — dependencies not installed. Run pip install -r requirements.txt again.

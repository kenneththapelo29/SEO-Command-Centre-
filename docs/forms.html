Google Forms

Connect form submission data to your SEO analysis.

Before using this fetcher you must enable two additional APIs in your Google Cloud project: Google Sheets API and Google Drive API. Without these the fetcher will fail with a permission error even if your service account is set up correctly.

Enable the required APIs

Go to console.cloud.google.com/apis/library and enable:

- Google Sheets API

- Google Drive API

These are separate from the Search Console and Analytics APIs you enabled during initial setup.

How it works

Google Forms automatically saves responses to a linked Google Sheet. The fetcher reads that sheet via the Google Sheets API using the same service account.

Setup

1. Open your Google Form → Responses tab → Link to Sheets → Create a new spreadsheet.

2. Share the spreadsheet with the service account email (Editor or Viewer access).

3. Add the sheet title to config.json:

"google_forms": [
  {"title": "Contact Form Responses", "output": "contact_leads.json"}
]

The title must exactly match the Google Sheet name.

Enable the Sheets API

Go to the Google Cloud API Library and enable Google Sheets API and Google Drive API on your project.

Fetch

python run_fetch.py --sources forms

Which organic keywords drove the most form submissions? What is my cost per lead from paid vs organic? Which pages convert traffic into leads?
